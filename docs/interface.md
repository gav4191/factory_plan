# Factory Plan: інтерфейс

## Тип додатку

Прийняте рішення: веб-додаток для локальної мережі.

Стек:

- Backend: FastAPI
- Frontend: HTML, CSS, vanilla JavaScript
- Database: SQLite
- Експорт Excel: пізніше

Сервер працює на Python і відкривається користувачами в браузері з локальної мережі.

## Основні сторінки

### `/`

Головний план по днях.

Показує зведення по датах і етапах:

- дата
- етап
- ємність годин
- заплановано годин
- перевищення годин
- кількість дверей

Користувач має бачити кнопку перерахунку плану і фільтр по датах.

### `/import`

Імпорт файлу дверей.

Функції:

- завантаження TSV/CSV файлу
- перевірка колонок
- попередній перегляд
- список помилок і попереджень
- імпорт у базу

### `/calendar`

Робочий календар.

Функції:

- список дат від сьогодні вперед
- ознака робочого/вихідного дня
- кількість робочих годин
- редагування дня
- генерація календаря на період

### `/priorities`

Ручне коригування черги.

Поля правила:

- тип правила: серія, номер замовлення, контрагент
- значення
- пріоритет
- коментар

### `/doors`

Таблиця дверей.

Функції:

- перегляд імпортованих дверей
- фільтри по контрагенту, замовленню, даті, типу, моделі
- перегляд стану етапів і годин

## Основні API

- `POST /api/import`
- `GET /api/doors`
- `GET /api/calendar`
- `PUT /api/calendar/{date}`
- `GET /api/priorities`
- `POST /api/priorities`
- `DELETE /api/priorities/{id}`
- `POST /api/plan/recalculate`
- `GET /api/plan`
- `GET /api/plan/load`

## Перша версія інтерфейсу

Перша версія має бути простою і робочою:

- без авторизації
- без складного дизайну
- з локальною SQLite базою
- з окремими сторінками для імпорту, календаря, правил черги, дверей і плану
