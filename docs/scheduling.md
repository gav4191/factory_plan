# Factory Plan: алгоритм планування

## Вхід алгоритму

Алгоритм отримує:

- двері
- операції дверей
- робочий календар
- правила ручної черги
- дату старту планування

## Побудова маршруту двері

Для кожної двері будується маршрут з потрібних незавершених операцій.

Операції:

- Заготовка
- Зварка
- Фарбування
- МДФ
- Зборка
- Упаковка

МДФ додається тільки якщо він має час.

## Залежності

Послідовність лінії A:

```text
Заготовка -> Зварка -> Фарбування
```

Лінія B:

```text
МДФ
```

Лінія C:

```text
Зборка -> Упаковка
```

Зборка залежить від:

- завершення лінії A
- завершення МДФ, якщо МДФ потрібен

Упаковка залежить від зборки.

Кожна наступна операція може бути не раніше наступного робочого дня після попередньої.

## Черга

Для кожної двері обчислюється ручний пріоритет.

Якщо правила немає, пріоритет дорівнює `0`.

Сортування:

1. ручний пріоритет
2. планована дата виготовлення
3. дата заказа
4. серія

Найконкретніше правило перемагає:

1. серія
2. номер замовлення
3. контрагент

## Денна ємність

Кожен робочий день має кількість годин.

Для кожного етапу на цей день:

```text
capacity_hours = work_calendar.hours
```

## Заповнення дня

Планувальник проходить по робочих днях від дати старту.

Для кожного етапу:

1. Шукає двері, для яких цей етап є наступним доступним.
2. Сортує їх за чергою.
3. Додає операції в день.
4. Якщо після додавання операції денна ємність перевищена, день для цього етапу закривається.

Операція не ділиться між днями.

## Вихід алгоритму

Детальний план операцій:

- дата
- етап
- лінія
- серія
- контрагент
- номер замовлення
- тип
- модель
- розмір
- кількість створок
- годин
- ручний пріоритет

Зведення завантаження:

- дата
- етап
- ємність годин
- заплановано годин
- перевищення годин
- кількість дверей
